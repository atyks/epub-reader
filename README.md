# AI-Driven EPUB Reader Project (Reconstruction Kit)

このプロジェクトは、生成系AIを活用して「自分好みの高品質なEPUBリーダー」を再構築・カスタマイズするための **ブループリント（設計図）リポジトリ** です。

ソースコードそのものを保守・管理するのではなく、**「優れた設計思想と解決済みのロジック（レシピ）」**を共有することで、誰でも最新のAI（ChatGPT, Claude, Gemini等）を使って同等の成果物を得ることを目的としています。

## 🌟 プロジェクトの目的
既存のオープンソースのEPUBリーダーには、以下のような課題がしばしば見られます：
- 内部CSSがアプリのUIを破壊する。
- 数千枚の画像を読み込むとメモリ不足でクラッシュする。
- コミックの見開き時に隙間ができる、または順序が狂う。
- 大規模な本で目次ジャンプがリンク切れを起こす。

本プロジェクトは、これらの「あるある」な問題をすべて解決済みのロジックとして `recipe.md` に集約しています。

## 🛠 コア・アーキテクチャ
本リーダーは、AIが最も理解しやすく、かつ堅牢に動作する以下の構造を採用しています。

- **Engine-Driven**: 全機能を統括する中心核（Engine）を置き、各モジュールを疎結合に保つ。
- **Strategy Pattern**: コミック（画像並列）とテキスト（巻物表示）の描画ロジックを分離。
- **Shadow DOM**: 書籍のCSSを完全にカプセル化し、アプリのUIを保護。
- **Zero-Gap Layout**: Flexboxを用いた、1pxの狂いもない見開き表示。

## 🚀 AIを使った再構築手順
このプロジェクトを自分のAIに渡し、以下の手順で開発を依頼してください。

1. **レシピの読み込み**:
   リポジトリ内の `recipe.md` をAIにアップロードし、「この設計図に従って、ElectronベースのEPUBリーダーを実装してほしい」と指示します。
2. **実装の依頼**:
   `recipe.md` に記載された「AIへの指示（プロンプト）のゴール」をそのままコピーして入力してください。
3. **カスタマイズ**:
   基本機能が完成したら、「ダークモードの色の調整」や「新しいショートカットキーの追加」などをAIに自由に依頼してください。
